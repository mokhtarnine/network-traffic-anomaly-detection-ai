The Problem

01 / 10
📈
Scale

Networks generate thousands of connections per second — manual inspection is impossible at this rate.

🎭
Variety

Attacks differ widely — DoS, Probe, R2L, U2R all produce distinct traffic signatures.

🏷️
No Labels

Real-world traffic arrives without pre-labeled attacks — supervised learning cannot be applied.

💡

Our approach: Use unsupervised K-Means clustering to discover behavioral groups in traffic data — no labels required at training time.

K-Means Clustering

05 / 10

Configuration

KMeans(
    n_clusters = k,      # 2 – 20 via slider
    random_state = 42,   # reproducible
    n_init = 10          # 10 starts → best
)

How it works

  • Randomly initialise k centroids
  • Assign each connection to its nearest centroid
  • Recompute centroids as cluster means
  • Repeat until inertia stops decreasing

🎯 Anomaly Detection Logic

  • After clustering, compare cluster contents with true labels
  • Compute attack ratio per cluster
  • Cluster with highest ratio → anomaly cluster
  • All its records flagged as is_anomaly = True
ratio  = attacks / total  # per cluster
anom   = argmax(ratio)    # auto-identified

⚡ Two Modes in the App

Train new — fits on uploaded data with chosen k

Pre-trained — loads saved k = 5 model instantly

Limitations & Future Work

09 / 10

⚠️ Current Limitations

K-Means requires k chosen in advance — wrong k leads to poor separation

One cluster ≠ one attack category — attacks can scatter across multiple clusters

NSL-KDD is academic — does not cover all modern attack patterns

Results are sensitive to preprocessing choices and random seeds

🚀 Planned Improvements

Auto k selection — Elbow, Gap statistic, Silhouette scan in UI

DBSCAN / Isolation Forest — no k required, density-based detection

Real-time traffic — stream live PCAP data into the pipeline

Cloud deployment — Streamlit Cloud or Docker container
